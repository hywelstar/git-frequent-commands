# Git Status 使用指南

## 适用场景

当你想知道当前仓库有哪些文件被修改、哪些内容已进入暂存区，以及自己当前位于哪个分支时，最先应该使用 `git status`。

## 基本命令

### 查看完整状态

```bash
git status
```

该命令会显示：

- 当前分支名称
- 工作区中已修改但未暂存的文件
- 已暂存、准备提交的文件
- 尚未跟踪的新文件

### 查看精简状态

```bash
git status --short
```

输出中常见的状态标记含义如下：

- `M`：文件已修改
- `A`：文件已添加到暂存区
- `D`：文件已删除
- `??`：未跟踪文件

### 同时查看分支与精简状态

```bash
git status -sb
```

这个命令适合日常快速检查，既能看当前分支，也能看简洁的文件变更列表。

## 常见使用场景

### 提交前检查

```bash
git status
```

确保只提交你真正想提交的文件。

### `git add` 后确认暂存结果

```bash
git status
```

确认需要提交的修改是否已经从 “Changes not staged” 进入 “Changes to be committed”。

### 清理前确认未跟踪文件

```bash
git status --short
```

适合在执行 `git clean`、切换分支或提交前快速确认目录状态。

## 注意事项

- `git status` 不会展示具体行级差异，如需查看内容改动，使用 `git diff`。
- 养成频繁执行 `git status` 的习惯，可以显著减少误提交的概率。
